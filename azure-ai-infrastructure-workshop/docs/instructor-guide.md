# Instructor guide

## Before delivery

1. Pin the tested Azure CLI, Anyscale CLI, KAITO add-on, CRD, preset, Python package, and container image versions.
2. Validate every command in a clean subscription or workshop tenant.
3. Preflight GPU quota and Anyscale on Azure access for every student/team.
4. Confirm the Anyscale cloud, project, compute configuration, identities, and storage integration already exist.
5. Prepare a low-cost path and an instructor demo path.
6. Capture expected outputs without including tenant IDs, subscription IDs, tokens, or endpoints.

## Do not teach in the Anyscale track

- Installing an Anyscale Operator with Helm
- Supplying operator tokens or identity IDs
- Maintaining operator values
- Building a self-managed Anyscale-on-AKS control/data plane

Those activities are outside this workshop's Anyscale on Azure scope.
